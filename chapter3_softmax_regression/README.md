## 模型结构

### Softmax回归模型

```python
ŷ = softmax(X·W + b)
```

**参数**：
- **W**: 权重矩阵，形状 `(784, 10)`
  - 784 = 28×28（图片像素数）
  - 10 = 类别数
- **b**: 偏置向量，形状 `(10,)`

### Softmax函数

```python
softmax(x_i) = exp(x_i) / Σ exp(x_j)
```

将输出转换为概率分布，所有类别概率之和为1。

### 交叉熵损失

```python
L = -log(ŷ[真实类别])
```

- 对于batch数据，计算每个样本的损失
- 反向传播时对batch求和

## 核心概念

### 1. Softmax回归 vs 逻辑回归

| 特性 | 逻辑回归 | Softmax回归 |
|------|---------|------------|
| 类别数 | 2（二分类） | K（多分类） |
| 输出 | 1个概率值 | K个概率值 |
| 激活函数 | sigmoid | softmax |
| 损失函数 | 二元交叉熵 | 交叉熵 |

### 2. 交叉熵损失的简化

**完整形式**（使用独热编码）：
```
L = -Σ(y_i × log(ŷ_i))
```

**简化形式**（代码实现）：
```python
L = -log(ŷ[真实类别索引])
```

因为独热编码中只有一个位置是1，其余都是0，所以只需要取真实类别对应的预测概率。

### 3. 小批量梯度下降

- **批量大小**: 256张图片/batch
- **更新频率**: 每个epoch更新 235次（60000÷256）
- **梯度计算**: 对batch中所有样本的损失求和后反向传播

### 4. 模型评估

**准确率计算**：
```python
accuracy = (预测正确的样本数) / (总样本数)
```

## 实现细节

### 数据加载

- 使用 `torchvision.datasets.FashionMNIST` 加载数据
- 使用 `DataLoader` 创建批量迭代器
- 自动随机打乱训练数据（`shuffle=True`）
- 使用4个进程并行加载数据

### 梯度下降流程

```python
for epoch in range(num_epochs):
    for X, y in train_iter:  # 遍历所有batch
        # 1. 前向传播
        y_hat = net(X)

        # 2. 计算损失（每个样本的损失）
        l = cross_entropy(y_hat, y)  # 形状: (batch_size,)

        # 3. 反向传播（对batch求和）
        l.sum().backward()

        # 4. 参数更新
        updater(batch_size)  # W = W - lr×W.grad/batch_size
```

## 常见问题

**Q: 为什么测试准确率比训练准确率低？**
A: 这是正常现象，因为模型在训练数据上见过这些样本，而测试数据是全新的。83.4%的测试准确率说明模型泛化能力良好。

**Q: 如何提高准确率？**
A: Softmax回归是线性模型，准确率有上限（~85%）。要提高准确率需要使用更复杂的模型，如：
- 多层感知机（MLP）
- 卷积神经网络（CNN）

**Q: 为什么不用独热编码？**
A: 代码使用类别索引而非独热编码，更节省内存和计算。数学上完全等价。
